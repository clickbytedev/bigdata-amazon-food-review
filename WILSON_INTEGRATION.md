# Wilson Helpfulness — Review Credibility Integration

## Background & Problem

The original pipeline used only a **star-rating penalty** (`get_weight`) as the sole weighting factor when computing `FinalSafetyScore`:

```
Weight           = get_weight(star_rating)   # 1★–2★ → 1.0 | 3★ → 0.8 | 4★–5★ → 0.1
FinalSafetyScore = DebertaScore × Weight
```

The problem: every review was treated equally in terms of **community credibility**. A review saying *"I got food poisoning"* with 0/0 helpful votes received the exact same weight as a nearly identical review with 95/100 helpful votes — even though the latter is statistically far more trustworthy.

---

## Solution: Wilson Lower Bound

The **Wilson Lower Bound** is a conservative statistical estimate of a true population proportion from a sample. It provides a statistically fair helpfulness score that avoids bias toward reviews with very few votes (e.g., 1/1 = 100% vs. 90/100 = 90%).

$$W = \frac{\hat{p} + \frac{z^2}{2n} - z\sqrt{\frac{\hat{p}(1-\hat{p})}{n} + \frac{z^2}{4n^2}}}{1 + \frac{z^2}{n}}$$

| Variable | Meaning |
|---|---|
| $\hat{p}$ | Raw helpfulness ratio (`HelpfulnessNumerator / HelpfulnessDenominator`) |
| $n$ | Total votes (`HelpfulnessDenominator`) |
| $z$ | 1.96 (95% confidence interval) |

Reviews with zero votes (`n = 0`) produce Wilson = 0.0. In this pipeline they are **floored to 0.5** so they are not penalized — just treated as neutral.

## Changes Made to `food_safety_pipeline_v2.ipynb`

### Change 1 — New Cell: Load Wilson Helpfulness Map

A new cell was inserted after the MongoDB setup cell. It:

1. Reads `outputs/wilson_helpfulness_output.csv` (generated by `Wilson_Helpfulness.ipynb`)
2. Builds `wilson_map = {amazon_id → wilson_score}` as a fast dict
3. Queries MongoDB for all `{_id, Id}` pairs and builds `oid_wilson_map = {mongo_oid → wilson_score}` for O(1) per-document lookup downstream
4. Defaults to **0.5** for any review not present in the CSV (no vote data)

```python
_df_wilson  = pd.read_csv('outputs/wilson_helpfulness_output.csv')
wilson_map  = dict(zip(_df_wilson['Id'].astype(int), _df_wilson['Wilson_Helpfulness']))

oid_wilson_map = {}
for doc in COL.find({}, {'_id': 1, 'Id': 1}):
    aid = doc.get('Id')
    oid_wilson_map[doc['_id']] = wilson_map.get(int(aid), 0.5) if aid else 0.5
```

### Change 2 — Compound Weight in Late-Weighting Step (Section 2 & Section 5)

**Before:**
```python
confirmed_zs['Weight'] = confirmed_zs['Score'].apply(get_weight)
```

**After:**
```python
confirmed_zs['WilsonScore'] = confirmed_zs['_id'].map(lambda oid: oid_wilson_map.get(oid, 0.5))
confirmed_zs['Weight']      = confirmed_zs['Score'].apply(get_weight) * (0.5 + 0.5 * confirmed_zs['WilsonScore'])
```

The formula `(0.5 + 0.5 × Wilson)` produces a credibility factor ranging from **0.5** (Wilson = 0, no votes) to **1.0** (Wilson = 1, highly trusted). Reviews with no votes are not eliminated, but community-validated reports receive full weight.

This change is applied in **two places**: the Zero-Shot pipeline (Section 2) and the Fine-Tuned pipeline (Section 5).

| Wilson Score | Credibility Factor | Effect on Weight |
|:---:|:---:|---|
| 0.00 (0 votes) | 0.50 | Weight halved |
| 0.50 | 0.75 | Weight reduced by 25% |
| 0.87 (90/100 votes) | 0.935 | Weight nearly unchanged |
| 1.00 | 1.00 | Full weight |

The compound weight formula combines both signals:

$$\text{Weight} = \underbrace{W_{\text{star}}}_{\text{rating penalty}} \times \underbrace{\left(0.5 + 0.5 \times W_{\text{Wilson}}\right)}_{\text{credibility factor}}$$

### Change 3 — `build_product_summary()` — Wilson-Weighted Bayesian Score (Section 8)

**Before:**
```python
raw = sum(SEVERITY_WEIGHT[sev] for sev in group['Severity']) / total_reviews
```

**After:**
```python
sev_w = group['Severity'].map(SEVERITY_WEIGHT).fillna(1)
raw   = (sev_w * group['WilsonScore'].fillna(0.5)).sum() / total_reviews
```

At the product level, each hazard report is now multiplied by its Wilson score before being summed into the `raw` hazard signal. Products whose hazard reports come from highly-voted, credible reviews will receive a higher `BayesianScore`. This prevents a product from being labelled *High Risk* solely because of many unverified, low-credibility complaints.

The Bayesian smoothing formula that follows remains unchanged:

$$\text{BayesianScore} = \frac{v \cdot R + m \cdot C}{v + m}$$

where $R$ is now the Wilson-weighted `raw` score, $v$ is total product reviews, $C$ is the global mean, and $m$ is the Q95 review-count prior.

### Change 4 — Excel Export Column Added (Section 7)

The `WilsonScore` column was added to both the `ZS_Confirmed` and `FT_Confirmed` Excel sheets so the per-review credibility score is visible alongside all other pipeline outputs.

## Impact Summary

| Aspect | Before | After |
|---|---|---|
| Weight dimensions | Star rating only | Star rating × community credibility |
| Review with 0/0 votes | Same weight as 90/100 review | Weight reduced by 50% (neutral floor) |
| Product BayesianScore | All hazard reports equal | Credible hazard reports weighted higher |
| Excel output | No Wilson column | `WilsonScore` column in Confirmed sheets |
